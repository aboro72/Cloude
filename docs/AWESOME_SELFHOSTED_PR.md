# awesome-selfhosted Pull Request — Submission Guide

## The entry (copy this exactly)

Add this line to the **Groupware** section of the awesome-selfhosted `README.md`,
in alphabetical order (after **Citadel**, before **Corteza**):

```
- [Cloude](https://github.com/aboro72/Cloude) - Intranet platform with file storage, real-time messenger, video calls, company news, team sites, kanban boards, and a visual page editor. ([Demo](https://cloudshare.aborosoft.com)) `MIT` `Python`
```

## Steps to submit the PR

1. **Fork** https://github.com/awesome-selfhosted/awesome-selfhosted
2. In your fork, open `README.md` and find the **Groupware** section
3. Add the entry above in alphabetical order (C → Cl → Cloude)
4. Commit with message: `Add Cloude`
5. Open a PR to `awesome-selfhosted/awesome-selfhosted` with:
   - **Title:** `Add Cloude`
   - **Body:** (see template below)

## PR body template

```
## Add Cloude

**URL:** https://github.com/aboro72/Cloude
**Demo:** https://cloudshare.aborosoft.com
**License:** MIT
**Language/Platform:** Python / Django

### Checklist
- [x] The software has an open source license (MIT)
- [x] Demo instance is available and live
- [x] The repo is public and actively maintained
- [x] The description is under 200 characters
- [x] The entry is in alphabetical order within the Groupware section
- [x] No self-promotion language in description

### Description
Cloude is a self-hosted intranet platform built with Django. It provides
SharePoint-equivalent features (file storage, team sites, messenger, news,
video calls, kanban boards, and a visual page editor) in a single deployable
application with a plugin system for extensibility.
```

## Notes

- awesome-selfhosted maintainers check that the demo is actually reachable.
  Make sure cloudshare.aborosoft.com is always online before submitting.
- They may ask for a screenshot in the PR discussion — have one ready.
- Review time is typically 1–4 weeks. The list gets ~50k GitHub stars, so
  a merged entry drives significant organic traffic.
